# 修复天气工具调用失败问题

## 问题分析

天气工具调用失败的根本原因：
1. **MCP 工具参数 schema 不明确**：通过 langchain_mcp_adapters 加载的 MCP 工具可能没有明确的参数 schema
2. **LLM 无法正确生成参数**：由于缺少明确的参数定义，LLM 在调用 maps_weather 工具时没有正确传递 city 参数
3. **工具内部读取 undefined 属性**：导致 "Cannot read properties of undefined (reading 'city')" 错误

## 解决方案

### 1. 创建工具包装器（在 mcp_tools.py 中）
为每个 MCP 工具创建包装器，添加明确的参数 schema：

```python
from langchain_core.tools import StructuredTool
from pydantic import BaseModel, Field

class WeatherToolInput(BaseModel):
    """天气查询工具输入参数"""
    city: str = Field(..., description="完整的城市名称，如 '北京市' 或 '深圳'")

def create_weather_tool_wrapper(mcp_tool: BaseTool) -> StructuredTool:
    """创建天气工具包装器"""
    async def weather_wrapper(city: str) -> str:
        # 调用底层 MCP 工具
        result = await mcp_tool.ainvoke({"city": city})
        return result
    
    return StructuredTool(
        name="maps_weather",
        description="查询指定城市的天气信息",
        func=None,
        coroutine=weather_wrapper,
        args_schema=WeatherToolInput
    )
```

### 2. 扩展到其他工具
为所有 MCP 工具创建类似的包装器：
- **TextSearchToolInput**：keywords, city
- **DirectionTransitToolInput**：origin, destination, city

### 3. 修改工具管理器
在 `MCPToolsManager` 中添加工具包装逻辑，确保所有工具都有明确的参数 schema。

### 4. 简化 Prompt
由于工具参数已经明确定义，可以进一步简化 Prompt，移除所有关于参数格式的说明。

## 实施步骤

1. 在 `mcp_tools.py` 中添加工具输入 schema 定义
2. 创建工具包装器函数
3. 修改 `MCPToolsManager` 的工具加载逻辑
4. 测试天气工具调用是否正常工作
5. 验证其他工具也能正常工作

## 预期效果

- LLM 能够根据明确的参数 schema 正确生成工具调用参数
- 天气工具调用不再出现 "Cannot read properties of undefined" 错误
- 系统整体更加健壮和可靠
