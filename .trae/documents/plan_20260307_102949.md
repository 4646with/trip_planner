# 修复 Planner 输出缺少 location 字段的问题

## 问题分析

从终端输出可以看到：
1. Planner 生成的 JSON 数据中，所有景点都缺少 `location` 字段
2. Pydantic 验证失败，提示 "Field required" 错误
3. 最终使用了安全版本（空的 days 列表），导致用户看不到任何行程建议

根本原因：
- `Attraction` 模型的 `location` 字段是必填的（`Field(...)`）
- `PLANNER_AGENT_PROMPT` 中没有包含 `location` 字段的示例
- LLM 不知道需要输出 `location` 字段

## 解决方案

### 方案一：将 location 字段设为可选（推荐）
优点：
- 更符合实际场景，LLM 可能无法准确生成经纬度坐标
- 提高系统的容错能力
- 避免因缺少 location 字段导致整个行程数据丢失

修改 `Attraction` 模型：
```python
location: Annotated[Optional[Location], BeforeValidator(clean_dict)] = Field(default=None, description="经纬度坐标")
```

### 方案二：修改 PLANNER_AGENT_PROMPT
在提示词中添加 `location` 字段的示例：
```json
{
  "name": "景点名称",
  "address": "景点地址",
  "location": {"longitude": 116.4, "latitude": 39.9},
  "visit_duration": 120,
  "description": "景点看点描述",
  "category": "景点",
  "ticket_price": 60
}
```

### 推荐方案：两者结合
1. 将 `location` 字段设为可选，提高容错能力
2. 在 `PLANNER_AGENT_PROMPT` 中添加 `location` 字段示例，引导 LLM 输出更完整的数据

## 实施步骤

1. 修改 `Attraction` 模型，将 `location` 字段设为可选
2. 修改 `PLANNER_AGENT_PROMPT`，添加 `location` 字段示例
3. 测试修复后的 Planner 输出

## 预期效果

- Planner 生成的数据能够通过 Pydantic 验证
- 用户能够看到完整的行程建议
- 系统更加健壮，即使缺少 location 字段也能正常工作
