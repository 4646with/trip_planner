# 多智能体项目重构方案

## 目标架构

```
backend/app/
├── agents/
│   ├── __init__.py
│   ├── schemas/
│   │   ├── __init__.py
│   │   └── state.py          # AgentState, RouteDecision
│   ├── prompts/
│   │   ├── __init__.py
│   │   └── agents.py         # 所有 Prompt 常量
│   ├── supervisor.py         # Supervisor 决策逻辑
│   ├── workers.py            # Worker Nodes
│   ├── graph/
│   │   ├── __init__.py
│   │   └── builder.py        # StateGraph 构建
│   └── utils/
│       ├── __init__.py
│       └── parsers.py        # JSON 提取和解析
├── main.py                   # 系统入口
└── map_agents.py (保留备份)  # 原文件备份
```

## 核心改进点

| 改进项 | 当前实现 | 重构后 |
|--------|----------|--------|
| State 管理 | 包含临时数据字段 | 精简 State，利用 Message History |
| Supervisor 输出 | 直接调用 LLM | 使用 `with_structured_output` 强制 Pydantic 输出 |
| Prompt 管理 | 硬编码在文件中 | 独立到 `prompts/agents.py` |
| 并发控制 | Semaphore 在类中 | 封装为中间件/工具层属性 |
| 代码组织 | 单文件 718 行 | 模块化拆分 |

## 关键改进说明

1. **State 精简**：移除 `attraction_data`, `weather_data` 等临时字段，改为通过 `messages` 传递上下文

2. **强类型 Supervisor**：使用 `with_structured_output(RouteDecision)` 强制输出 Pydantic 模型

3. **Prompt 独立**：所有 Prompt 移到 `prompts/agents.py`，支持配置化

4. **保持兼容性**：对外接口 `get_map_agents_system()`, `initialize_map_agents_system()`, `cleanup_map_agents_system()` 保持不变

## 重构步骤

1. 创建新目录结构
2. 迁移 `schemas/state.py`（状态定义）
3. 迁移 `prompts/agents.py`（提示词）
4. 实现 `utils/parsers.py`（解析工具）
5. 实现 `supervisor.py`（决策中心）
6. 实现 `workers.py`（工作节点）
7. 实现 `graph/builder.py`（图构建）
8. 实现 `main.py`（系统入口）
9. 测试验证
10. 删除旧文件

## 保证事项

- ✅ 代码模块化，易于维护
- ✅ 对外接口不变，不影响现有调用
- ✅ 保留原有业务逻辑
- ✅ 引入更好的类型安全
